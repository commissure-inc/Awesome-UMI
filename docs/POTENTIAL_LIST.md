## Priority legend

| Priority | Meaning |
|----------|---------|
| **P1** | Direct UMI lineage or explicit UMI-style extension |
| **P2** | Wearable / exoskeleton / tactile demonstration interfaces |
| **P3** | Handheld gripper or multimodal collection devices |
| **P4** | Teleoperation platforms — useful reference, may be borderline for scope |

## How to promote an entry

1. Verify primary sources (paper, project page, GitHub).
2. Check scope against [ADD_DEVICE.md](https://github.com/commissure-inc/UMI-Data-Collection/blob/main/docs/ADD_DEVICE.md).
3. Submit via the live site *Contribute* tab or [Add a device issue](https://github.com/commissure-inc/Awesome-UMI/issues/new?template=add_device.yml).
4. Remove the row from this list once merged into `umi_devices_data.json`.
